# CLAUDE.md - AI/Health/Conference Daily Intelligence

> 继承全局规则。每次会话开始先读本文件、`PROGRESS.md` 和 `MEMORY.md`。

## 这个项目要解决什么问题

- 核心问题：每天自动收集 AI、医疗健康资讯和国内会议信息，形成可复核、可追踪的情报日报。
- 本期范围：设计并实现一个轻量脚本，优先支持 GitHub Actions 定时运行，也支持本地运行后落 Markdown/HTML/JSON 文件。
- 不做：不抓取登录后内容，不绕过反爬，不采集个人隐私数据，不做实时舆情大盘，不做复杂 Web 后台。
- 成败标准：每日可稳定生成一份结构化日报；每条信息保留来源链接、发布时间、分类、摘要和去重标识；失败源不影响整体输出。

## 项目概况

- 类型：信息采集与日报自动化脚本。
- 当前阶段：需求/技术评审。
- 技术倾向：Python 脚本 + 配置信源 + 本地文件输出 + 可选 GitHub Actions。

## 项目特有规范

- 信源优先级：官方 RSS/公开 JSON > 官方公开列表页 > 搜索/RSSHub 兜底 > 暂不接入。
- 合规边界：不使用账号登录态、cookies、私有 API token 抓内容；对 robots、访问频率和站点条款保持保守。
- 内容原则：资讯只做摘要和链接索引，不全文转载；医疗健康内容必须标注“资讯摘要，非医疗建议”。
- 会议信息：必须尽量结构化到名称、时间、地点、主办方、报名/官网链接、领域标签。
